Server Startup Issues

Issue: "Port 3000 is already in use"
Symptoms:
Error: listen EADDRINUSE: address already in use :::3000
Solutions:

1. Find and kill the process using port 3000:

# Windows netstat -ano | findstr :3000 taskkill /PID <process_id> /F # Alternative: Use different port # Modify index.js to use different port const PORT = 3001;

2. Check for other applications:

  • Close other development servers
  • Check for running Node.js processes
  • Restart your computer if necessary

Module Loading Issues

Issue: "Cannot find module" errors
Symptoms:
Error: Cannot find module 'http' Error: Cannot find module 'fs'
Solutions:

1. Verify Node.js installation:

node --version npm --version

2. Reinstall Node.js if needed:

  • Download from nodejs.org
  • Choose LTS version for stability

3. Check file paths:

  • Ensure you're in the correct directory
  • Verify all files are present

JSON Processing Issues

Issue: JSON files not loading or parsing errors
Symptoms:
SyntaxError: Unexpected token in JSON Failed to fetch 1.json
Solutions:

1. Validate JSON syntax:

# Use online JSON validator or node -e "console.log(JSON.parse(require('fs').readFileSync('1.json', 'utf8')))"

2. Check file encoding:

  • Ensure files are saved as UTF-8
  • Remove any BOM (Byte Order Mark) characters

3. Verify file permissions:

  • Ensure files are readable
  • Check file ownership and permissions

üåê Browser Compatibility Issues

Elements API Component Not Loading

Symptoms:
  • Blank page or missing documentation interface
  • Console errors about custom elements
Solutions:

1. Check browser compatibility:

  • Use modern browsers (Chrome 60+, Firefox 63+, Safari 10.1+)
  • Enable JavaScript if disabled

2. Verify web components support:

// Check in browser console if ('customElements' in window) { console.log('Web Components supported'); } else { console.log('Web Components not supported'); }

3. Load polyfills if needed:

<!-- Add before other scripts --> <script src="https://unpkg.com/@webcomponents/webcomponentsjs@^2/webcomponents-loader.js"></script>

CORS Errors When Testing API

Symptoms:
Access to fetch at 'api/endpoint' from origin 'http://localhost:3000' has been blocked by CORS policy
Solutions:

1. Add CORS headers to server:

// In index.js res.writeHead(200, { 'Content-Type': contentType, 'Access-Control-Allow-Origin': '*', 'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE', 'Access-Control-Allow-Headers': 'Content-Type, Authorization' });

2. Use browser flags for development:

# Chrome with disabled security (development only) chrome.exe --user-data-dir=/tmp/chrome_dev_session --disable-web-security

Slow Page Loading

Symptoms:
  • Long loading times
  • Unresponsive interface
  • High memory usage
Solutions:

1. Optimize JSON file size:

// Minify JSON files const minified = JSON.stringify(JSON.parse(jsonString));

2. Enable compression:

// Add to index.js const zlib = require('zlib'); // Compress responses if (req.headers['accept-encoding']?.includes('gzip')) { res.writeHead(200, { 'Content-Type': contentType, 'Content-Encoding': 'gzip' }); res.end(zlib.gzipSync(data)); }

3. Implement caching:

// Add cache headers const cacheTime = 3600; // 1 hour res.writeHead(200, { 'Content-Type': contentType, 'Cache-Control': `max-age=${cacheTime}` });
